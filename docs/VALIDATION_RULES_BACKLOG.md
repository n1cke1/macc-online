# Backlog: семантические правила валидации (L2) — отдельная таска

> Запаркованные правила валидации, отложенные от роадмапа авторинга
> (`docs/MEASURE_AUTHORING_ROADMAP.md`). Это **семантический слой L2** — согласованность
> величин, классификации, честность бейджа `eligibleForModel`. Делается отдельно, чтобы
> не смешивать с готовностью движка/библиотеки/гида.
>
> Контекст и полный разбор оси MECE — в `ISSUES_AUDIT_VALIDATION.md` (Issue A).

## Состав

### Из `ISSUES_AUDIT_VALIDATION.md` (Issue A, семантический хвост)
- **A1.3 · Когерентность классификации** — продукт ↔ сектор ↔ пул должны быть согласованы
  (промтепло в угольном электропуле + электрический продукт на тепловой мере — прошло «зелёным»).
- **A1.5 · Обязательные коридоры** — `factor`/`economics` opt-in: нет `reference_ref` → проверки
  правдоподобия нет, молча. Сделать коридор обязательным для чисел, материально влияющих на MAC.
- **A1.6 · Честный бейдж** — `eligibleForModel` либо включает неавтоматизированные правила
  (`serviceUnitMatch`, `doubleCountReduction`), либо показывает «N/2 правил целостности не проверены».
- **A2 · Ортогональность осей / перегрузка имён** — панели переэкспрессируют чеки
  (`limit`/`pool` в двух поверхностях); «economics» и «provenance» означают по две разных вещи.

### Из `macc-issues.md` (валидаторные пункты)
- **B4 · `pool: warn` vs eligibility** — гид говорит «удерживает от eligibility», код не блокирует
  (clip → `displaced`). Привести доку к поведению или наоборот.
- **B5 · `limit` обязателен** — *уже FIXED в коде* (в `GATING_CHECKS`); оставлено как точка сверки доки.
- **C8 · Несведённый EF угля 135.3 ≠ 88.3** — нет owner-coherence чека
  (emissions ≈ generation × ef). Это и есть долг **materials-coherence lock** из памяти проекта.
- **C11 · `economics`-чек тихо `na`** — отсутствие коридора показывать как `incomplete`/⚠, не `na`.
  Частично пересекается с A1.5.
