**Статус:** ошибки данных исправлены; целевая структура утверждена. Дальше — только рефакторинг
с сохранением выходов **бит-в-бит**.
**Файл:** `MACC_KZ_29052026_rev.xlsx`. **Контекст:** `CLAUDE.md`, `~/.claude/plans/macc-twinkly-dove.md`.
**Принципы:** только `IF/PV/SUM` + арифметика, ссылки между листами, статическое ядро.

---

## 1. Целевая архитектура

| уровень | что | где живёт |
|---|---|---|
| **L0** | Глобальные допущения (цены, EF, удел. CAPEX, WACC) | `MACC` 36–57, `C2` |
| **L1** | Допущения по мероприятию (КПД, КИУМ, CF, срок, мощность, доли, % ПНР/ТОиР) | **новый лист «Допущения»** |
| **L2–L4** | Расчёт → сборка затрат → выходы в MACC | `Расчёты` (шаблонные блоки) |
| **L5** | Кривая (NPV, disc CO₂, MAC) | `MACC` I/J/K |
| **BASE** | Базлайны секторов | `Выбросы` |

**L1 — отдельный длинный (tidy) лист**, строка = `(код_мероприятия, ключ)`:

```
код_мероприятия · ключ · подпись_RU · подпись_EN · значение · единица · источник · min · max · шаг · тег
```

Почему отдельный лист, а не теги на месте: «добавить проект» = дописать строки в конец реестра +
шаблонный блок в конец `Расчётов` (геометрия блоков жёсткая — вставка в середину опасна);
уникальность `(код, ключ)` делает рассогласованные инпуты невыразимыми; онлайн-инструмент читает
реестр как данные почти без ETL.

В `Расчётах` строка-инпут сохраняет подпись и становится **ссылкой** (`=Допущения!$E$23`) — значение
задаётся в реестре, но видно на месте.

## 2. Фазы (по порядку)

### A — Теги уровня в `Расчётах` (MVP, формулы не трогаются)

Столбец-тег на каждой содержательной строке:

| тег | смысл |
|---|---|
| `IN-G` | глобальный инпут (ссылка на L0) |
| `IN-L` | локальный инпут (L1) — редактируемое допущение |
| `BASE` | базлайн (`=Выбросы!…`) |
| `CALC` | производная величина |
| `OUT`  | выход в таблицу MACC |

Детерминированный ETL вместо угадывания подписей; ни одна формула не меняется. Структурный минимум
для публикации.

### C — Реестр «Допущения»

Вынести 158 литералов L1 в реестр (схема выше), перелинковать строки-инпуты в `Расчётах`. Попутно —
извлечь смысловые коэффициенты-доли из формул (`*0.3` ПНР, `*0.01` ТОиР, `0.72`, `0.95`…); чистые
конвертеры единиц (`44/12`, `1000`, `10^6`, `8760`) оставить инлайн как физику. Диапазоны `min/max/шаг`
заполнять только для рычагов под слайдеры (~30–40: цены, КИУМ, CF, доли, сроки); остальным — пусто.

### D — Унификация шаблона блока

Единый порядок секций во всех 26 блоках → добавление проекта = инстанцирование шаблона из реестра,
ревью — построчным diff'ом.

## 3. Верификация

Все фазы — рефакторинг: выходы `I/J/K` сохраняются **бит-в-бит** против кэша Excel.
После каждого шага: `py scripts/etl.py --check` → `py scripts/etl.py` → перегнать golden → коммит.
Перестройка меняет `modelVersion`/fingerprint — нормально.

ETL переходит с позиционно-эвристического на метаданно-управляемый (тег-колонка + реестр) — уходят
`_PHYS_UNIT`/`_BAD_UNIT`/`extract_breakdown`/`build_physical_items`.
